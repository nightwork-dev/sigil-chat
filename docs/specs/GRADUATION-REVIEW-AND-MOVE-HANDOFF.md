## Review boundary

simulated example is not an acceptance surface.

The maintainer-managed review lane needs to close two decisions:

1. T1/T2/T5 correctness, using `T1-T2-T5-REVIEW-PACKET.md` and its live-host
   regressions; and
2. the released Sigil Agent extraction, especially neutral contracts,
   provider boundaries, Eve translation, React Query reconciliation, registry
   presentation, and authenticated Gonk mounting.

Reviewers should return `APPROVE` or severity-ranked findings. Do not mark the
ledger done from test output alone.
